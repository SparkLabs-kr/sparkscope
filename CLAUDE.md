# SparkScope — Claude 협업 규칙

## 🔒 Git 워크플로우

### 브랜치 정책

**이수(isujang-ctrl):**
- ✅ main에서 직접 작업 및 커밋
- ❌ 브랜치 생성 불필요
- ✅ 모든 변경사항 직접 main에 푸시

**인턴:**
- ❌ main에 직접 커밋 금지
- ✅ **모든 변경(문서 포함)을 반드시 브랜치 → PR로만** 진행
- ✅ PR 병합은 이수 승인 필요
- ❌ 다른 사람의 브랜치로 push 금지
- ❌ force push 절대 금지

### 브랜치 네이밍
- 개인 작업: `이수브랜치` (담당자가 이수일 때)
- 협업 기능: `feature/기능명` 또는 `fix/버그명`

### 커밋 메시지
- 한글 또는 영문 모두 가능
- 형식: `[type]: 설명`
  - `feat:` 새 기능
  - `fix:` 버그 수정
  - `docs:` 문서 변경
  - `refactor:` 코드 정리
  - `test:` 테스트 추가

## 🚨 Pre-commit 보안 훅

이 저장소는 API 키, 토큰, 시크릿이 커밋되는 것을 자동으로 차단합니다.

```bash
npm install  # 자동으로 husky 활성화
git commit -m "메시지"  # 자동으로 보안 검사 실행
```

시크릿이 감지되면 커밋이 중단됩니다. 파일에서 민감한 정보를 제거 후 다시 시도하세요.

**우회가 필요한 경우** (권장하지 않음):
```bash
git commit --no-verify
```
