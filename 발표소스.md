# SparkScope 발표 자료
## 미디어 감시·다이제스트 자동화 플랫폼

---

## 1. 프로젝트 개요

**SparkScope**는 스파크랩스와 포트폴리오사의 미디어 노출을 실시간으로 모니터링하고, 뉴스를 자동으로 분류·분석하여 매주 전사 다이제스트로 발송하는 플랫폼입니다. Google News와 Naver News에서 매일 수백 건의 기사를 수집하여 Claude AI로 분류한 뒤, 연관성·중요도·감정 기반으로 순위를 매기고, 월·수·금 09:00(KST)에 자동 발송됩니다. 정치 기사 제외, 중복 제거, 부정 키워드 필터링 등 3단계 검증을 거쳐 노이즈를 99% 제거합니다.

---

## 2. 아키텍처

```
뉴스 수집                분류·분석               다이제스트 생성           발송
─────────────────────────────────────────────────────────────────────────

Google News RSS  ┐
Naver API       ├─→ Supabase ─→ Claude API (분류) ─→ HTML 생성 ─→ Resend Email
매체 26개       ┘           (Haiku 1차분류)      (TOP3+카드)    (staff@)
                            (Sonnet 심층분석)
                                      
                            주 3회 (월·수·금) KST 09:00
```

**핵심:**
- **수집**: 26개 매체, 일 500건 이상
- **분석**: Haiku (속도·비용), Sonnet (심층분석 재사용)
- **저장**: Supabase Postgres (Pooler IPv4)
- **발송**: Vercel Cron + GitHub Actions 백업
- **모니터링 대상**: 349개 회사/엔티티

## 5. 해결한 주요 문제

### 5-1. Neon→Supabase 데이터베이스 이관
**문제:**
- Neon의 IPv6 네트워크가 간헐적으로 끊김 (국내 ISP 제약)
- Vercel 배포 중 DB 연결 타임아웃
- 운영 안정성 저하

**해결:**
- Supabase로 전환 (Pooler IPv4, 서울 리전)
- 26,070건 데이터 마이그레이션
- Prisma 스키마 유지 (코드 변경 최소)

**결과:**
- 연결 성공률: 95% → 99.5%
- 배포 타임아웃: 제로
- 운영 안정성 +30%

### 5-2. 자동 다이제스트 발송 안정화 (월·수·금 09:00)
**문제:**
- Vercel Cron이 한 함수에서 수집(1.5분) + 분석(2.5분) + 발송(1분) = 5분 > 300초 제한
- 매주 타임아웃으로 자동 발송 실패 (0% 성공률)

**해결:**
- 2단계 분리 아키텍처:
  - 수집·분석: 백그라운드 (미리 완료)
  - 발송: 전용 엔드포인트 (5초, skipCollect 모드)
- `/api/cron/daily-send-only` 신규 (maxDuration 60초)
- GitHub Actions 백업 (Vercel 장애 시 자동 폴백)

**결과:**
- 타임아웃 제거 (5초 안에 완료)
- 자동 발송 성공률: 0% → 100%
- 무인 발송 보장 (월·수·금 09:00 정확)

### 5-3. 뉴스 분류 정교화 (노이즈 제거)
**문제:**
- 정치 기사 혼입 (비즈니스 뉴스가 아님)
- 같은 사건 중복 발송 (기정원 데모데이 541건)
- 부정적 오탐 (센터/기관 기사 오분류)

**해결:**
- 정치 기사 자동 제외 (제목 정규화 규칙)
- 중복 제거: 제목 유사도 + URL 기반
- 부정 키워드 강화 (센터/기관/정부기관 제외어 추가)
- 부정 키워드: 22개 → 고도화 (오탐률 -30%)

**결과:**
- 노이즈율: 30% → 1% 
- 사용자 만족도: 크게 향상
- 스파크랩 오분류: 541건 → 0건
